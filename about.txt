Сайт для поиска угнанных велосипедов.
К сожалению, из-за курса доллара все меньше людей могут себе позволить приобрести новый хороший и,
следовательно, дорогой байк в магазине. Многие покупают на он-лайн барахолках, где часто
выставляют "не свои" велосипеды.

----------Идея------------------------------------------------------------------------
 
У человека украли велосипед. Он запостил объявление с макимально развернутым описанием. Другой человек,
покупая байк или какую-то комплектацию на барахолке, проверяет по данному сайту, не в розыске ли
он. Если да - сообщает владельцу, что его велосипед продается там-то человеком с такими-то контактами.
Таким образом, человек, покупающий на форуме/доске не купит краденное (что потом может иметь
негативные последствия) и поможет владельцу вернуть свой байк.

---------Что реализовано на сайте---------------------------------------------------------------------

Регистрация. 
Подать объявление можно только зарегистрировавшись.Регистрация с подтверждением по почте.
При успешном заполнении формы на и-мейл пользователю отправляется ссылка со случайным образом
сгенерированным токеном. Этот же токен записывается в базу в данные юзера. Когда он переходит по
ссылке, скрипт ищет в базе юзера с таким токеном и, если находит, активирует аккаунт.

Авторизация.
Обычная авторизация по логину и паролю. Есть опция "Запомнить меня", работает следующим образом.
Если поставлен чек-бокс, в базу в информацию юзера пишется идентификатор сессии (случайный токен),
он же пишетя юзеру в куки. Также в куки пишется id юзера.
При повторном заходе на сайт, если php-сессии к тому времени уже не существует, пытаемся авторизоваться
через куки - считываем id юзера и id сессии. Проверяем по базе, действительно ли юзеру с таким id
соответствует такой id сессии. Если да - авторизовываем. Нет - удаляем куки. 
Когда пользователь разлогинивается, также удаляем данные куки. 

Восстановление доступа.
Забывчивый пользователь вводит свой логин или почту, после чего, в случае нахождения данного
значения в базе, на имейл высылается ссылка, содержащая md5-хеш, который также записывается в БД
пользователю. Когда он переходит по ссылке, проверяется наличие юзера с таким хешем в соответствующем
поле, после чего, в случае успеха, показывается форма для ввода нового пароля. 

Добавление объявления.
Авторизованный юзер может добавить объявление, состоящее из фото велосипеда, названия, даты кражи, 
города, свободного описания, серийного номера и моделей компонентов, из которых состоит велосипед
(для каждой комплектующей свое поле). Также подключены гугл-карты. На карте можно указать точное место
кражи с помощью маркера. Когда открывается страница создания объявления, определяется город пользователя
по IP и на карте по умолчанию показыватся этот город.
При загрузке фото создаются 3 версии: оригинальная, уменьшенная и маленькая квадратная превьюшка.
Для создания превьюшки скрипт сам определяет ориентацию фото и обрезает соответственно только верх/низ
или только бока, оставляя центр, чтоб получилось квадратное изображение независимо от ориентации исходника.
Превьшка используется на главной, она же страница поиска, при показе всех объявлений и на странице
объявления. Во втором случае, при наведении на нее, внизу появляется кнопка для открытия оригинального
(неуменьшенного) изображения в новом окне, а при клике на саму превьюшку открывется модальное окно
с уменьшенной до заданного размера фоткой.

Редактирование объявления
Реализовано еще не полностью. Что уже есть.
Редактирование происходит не на отдельной странице, а прямо на странице объявления. Когда открывается
объявление,если пользователь авторизован и если он является автором объявления, при наведении мышки на
любое заполненное поле комплектации, на этом поле появляется кнопка, активирующая редактирование этого
поля (input с кнопкой отправки). Обновление происходит на лету, без перезагрузки страницы.
Также присутствует возможность добавить недостающие поля комплектации: происходит проверка, какие поля
уже заполнены для данного объявления, и присылается (без перезагрузки страницы) форма с недостающими
полями. Отправка и внесение в БД этих данных еще не сделана, так же, как обновление фото и описания.

На главной показаны превьюшки объявлений с фото и основной информацией, при клике на которые открывается
страница объявления. Присутствует постраничная навигация и поиск, который ведется по полям таблицы
объявлений, содержащим значащую для пользователя информацию (город, компоненты, название велосипеда).
Результаты поиска выводятся в таком же формате, как все объявления на главной.
